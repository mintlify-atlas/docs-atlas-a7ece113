## List Club Applications

<CodeGroup>
```bash GET /api/clubs/{club_code}/applications/
List all applications for a club
```
</CodeGroup>

### Authentication
Required - must be club officer or owner

### Path Parameters

<ParamField path="club_code" type="string" required>
  Club code identifier
</ParamField>

### Response Fields

<ResponseField name="id" type="integer" required>
  Application ID
</ResponseField>

<ResponseField name="name" type="string">
  Application name
</ResponseField>

<ResponseField name="description" type="string">
  Application description (HTML)
</ResponseField>

<ResponseField name="application_start_time" type="datetime">
  When application opens
</ResponseField>

<ResponseField name="application_end_time" type="datetime">
  When application closes
</ResponseField>

<ResponseField name="result_release_time" type="datetime">
  When results are released
</ResponseField>

<ResponseField name="is_active" type="boolean">
  Whether application is currently active
</ResponseField>

<ResponseField name="external_url" type="string">
  External application URL
</ResponseField>

<ResponseField name="acceptance_email" type="string">
  Email template for accepted applicants
</ResponseField>

<ResponseField name="rejection_email" type="string">
  Email template for rejected applicants
</ResponseField>

<ResponseField name="committees" type="array">
  Array of application committees
</ResponseField>

<ResponseField name="questions" type="array">
  Array of application questions
</ResponseField>

### Example Request

```bash
curl https://pennclubs.com/api/clubs/pppjo/applications/ \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### Example Response

```json
[
  {
    "id": 42,
    "name": "Fall 2024 Recruitment",
    "description": "<p>Join our team!</p>",
    "application_start_time": "2024-09-01T00:00:00Z",
    "application_end_time": "2024-09-15T23:59:59Z",
    "result_release_time": "2024-09-20T18:00:00Z",
    "is_active": true,
    "external_url": "https://pennclubs.com/club/pppjo/application/42",
    "committees": [
      {"id": 1, "name": "Engineering"},
      {"id": 2, "name": "Design"}
    ],
    "questions": [
      {
        "id": 100,
        "question_type": 1,
        "prompt": "Why do you want to join?",
        "word_limit": 200,
        "precedence": 0
      }
    ]
  }
]
```

## Create Application

<CodeGroup>
```bash POST /api/clubs/{club_code}/applications/
Create a new application for the club
```
</CodeGroup>

### Authentication
Required - must be club officer or owner

### Path Parameters

<ParamField path="club_code" type="string" required>
  Club code identifier
</ParamField>

### Request Body

<ParamField body="name" type="string" required>
  Application name
</ParamField>

<ParamField body="description" type="string">
  Application description (HTML)
</ParamField>

<ParamField body="application_start_time" type="datetime" required>
  When application opens (ISO 8601 format)
</ParamField>

<ParamField body="application_end_time" type="datetime" required>
  When application closes
</ParamField>

<ParamField body="result_release_time" type="datetime" required>
  When results are released
</ParamField>

<ParamField body="is_active" type="boolean">
  Whether application is active (default: false)
</ParamField>

<ParamField body="acceptance_email" type="string">
  Email template for accepted applicants (Jinja2 template)
</ParamField>

<ParamField body="rejection_email" type="string">
  Email template for rejected applicants (Jinja2 template)
</ParamField>

### Example Request

```bash
curl -X POST https://pennclubs.com/api/clubs/pppjo/applications/ \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Spring 2025 Recruitment",
    "description": "<p>Join our amazing team!</p>",
    "application_start_time": "2025-01-15T00:00:00Z",
    "application_end_time": "2025-02-01T23:59:59Z",
    "result_release_time": "2025-02-10T18:00:00Z",
    "is_active": true
  }'
```

## Submit Application

<CodeGroup>
```bash POST /api/clubs/{club_code}/applications/{application_id}/submissions/
Submit an application
```
</CodeGroup>

### Authentication
Required

### Request Body

<ParamField body="committee" type="integer">
  Committee ID to apply to
</ParamField>

<ParamField body="responses" type="array" required>
  Array of question responses
</ParamField>

Each response should have:
- `question`: Question ID
- `text`: Response text (for free response/short answer)
- `multiple_choice`: Multiple choice option ID (for multiple choice)

### Example Request

```bash
curl -X POST https://pennclubs.com/api/clubs/pppjo/applications/42/submissions/ \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "committee": 1,
    "responses": [
      {
        "question": 100,
        "text": "I have experience with Python and Django..."
      },
      {
        "question": 101,
        "multiple_choice": 5
      }
    ]
  }'
```

## Send Decision Emails

<CodeGroup>
```bash POST /api/clubs/{club_code}/applications/{application_id}/send_emails/
Send acceptance/rejection emails to applicants
```
</CodeGroup>

### Authentication
Required - must be club officer or owner

### Request Body

<ParamField body="email_type" type="object" required>
  Email type to send (id: "acceptance" or "rejection")
</ParamField>

<ParamField body="allow_resend" type="boolean">
  Allow re-sending to already notified applicants (default: false)
</ParamField>

<ParamField body="dry_run" type="boolean">
  Preview emails without sending (default: false)
</ParamField>

### Example Request

```bash
curl -X POST https://pennclubs.com/api/clubs/pppjo/applications/42/send_emails/ \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "email_type": {"id": "acceptance"},
    "allow_resend": false,
    "dry_run": true
  }'
```
